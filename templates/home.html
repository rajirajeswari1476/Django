Welcome to Palle Dashboard

1. Introduction

The Palle Admin Project is a role-based access control (RBAC) system built with Django for managing employees and students. It supports two user types:

  • Admin – Full permissions
  • Sales – Limited access

2. System Components & Workflow

2.1 User Roles & Permissions

  • Admin (Superuser/Full Control)
    • View, add, update, and delete employees and students
    • Full access to Admin Dashboard
    • Can assign a student to any employee via the added_by dropdown
  • Sales (Restricted Access)
    • Can add new students and view student list (read-only)
    • Cannot modify/delete students or view employees
    • added_by is auto-filled with their own username

2.2 Authentication & Login Flow

  • Superuser Creation: Created using Django’s createsuperuser command
  • Role-Based Redirect:
    • Admins → Admin Dashboard
    • Sales → Sales Dashboard
  • Welcome Message: Displays "Welcome, [Username]"

3. Functionality

3.1 Admin Dashboard Features

  • Navbar Options:
    • Employee List (View/Add/Delete)
    • Student List (View/Edit/Delete)
    • Logout
  • Student Creation: added_by dropdown with all employees

3.2 Sales Dashboard Features

  • Navbar Options:
    • Add Student (Form submission)
    • View Student List (Read-only)
    • Logout
  • Student Creation: added_by auto-filled with logged-in username

4. Technical Highlights

  • Django ORM: Used for all database operations such as .filter(), .get(), .create(), and .update().
  • Function-Based Views (FBVs): All logic implemented using standard Django view functions for better control and customization.
  • ModelForms: Used to create forms from models for validation and clean input handling.
  • Role Handling Logic: Access control handled directly in FBVs using request.user checks and conditions.
  • Django Admin Panel: Extended for backend data viewing with list displays, filters, and inline edits.
  • Session & Message Framework: Sessions used for login tracking and success/error messages shown using Django’s messaging system.

5. Security & Data Integrity

  • Role-Based Access: Sales users restricted from accessing admin-only features
  • Field Control: added_by is enforced and protected from tampering
  • CSRF Protection: All POST forms use {% csrf_token %}
  • Session Handling: Proper login/logout with redirects and protections

6. Technology Stack

  • Backend: Python, Django (FBVs, ORM, authentication)
  • Frontend: Django Templates, HTML, Bootstrap, CSS
  • Database: MySQL with normalized schema and constraints

7. Summary

  • Admin users get full CRUD access
  • Sales users have limited access (add + read)
  • Dynamic dashboards based on role
  • Clean, secure handling of student assignments via added_by
  • Used Django ORM and Function-Based Views for all logic
  • Forms built using ModelForms for validation
  • Admin site used for backend data management
